Section 8 — Verified Reality Claims

  1. Real execution can be denied in production when no valid Decision ID is present.
  2. The identical execution succeeds immediately when a valid, scoped, unexpired Decision ID is provided.
  3. Execution produces independent Proof-of-Transfer confirming that reality changed only as authorized.
  4. Authority expires automatically and cannot be silently reused.
  5. Non-execution (DENY with no side effects) is observable and logged as positive evidence of enforcement.

Client identifiers and Decision IDs are intentionally redacted.

Section 9 — Public Verification Path

What can be verified

  • Execution is impossible without a valid Decision ID.
  • The same execution succeeds when valid authority is presented.
  • Outcomes are independently verifiable via Proof-of-Transfer.
  • Authority expires and cannot be silently reused.

Verification procedure

  • Denial test: attempt a real execution without a Decision ID → must DENY; no real-world change.
  • Allowance test: retry the identical execution with a valid Decision ID → must ALLOW; execution completes.
  • Proof-of-Transfer: verify independent evidence of the authorized change.
  • Expiry re-denial: after expiry or consumption, retry → must DENY again.

What cannot be verified (by design)

  • Decision Registry contents
  • Decision IDs or credentials
  • Enforcement logic or thresholds
  • Internal bindings or integrations
  • Human deliberation or intent

Verification guarantee: If any execution succeeds without a valid Decision ID, MindShift is not present or not enforced.
